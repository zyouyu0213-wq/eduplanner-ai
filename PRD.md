**Product Owner: Youyu**

## 1. 产品概述

### 1.1 产品定义

EduPlanner AI 是一款面向留学咨询顾问的 AI 辅助工具。顾问输入学生背景信息和专业方向，AI 基于结构化的方法论自动生成个性化的 Personal Tag（PT）、Personal Goal（PG）以及 10 项活动规划方案，顾问审核调整后导出完整的签约方案文件。

### 1.2 产品信息

| 字段 | 内容 |
|------|------|
| 产品名称 | EduPlanner AI |
| 产品类型 | B端 SaaS 工具（Web 应用） |
| 版本 | V1.0 MVP |
| 目标用户 | 留学咨询顾问（国际高中方向） |
| 核心价值 | 将 2-3 小时的方案设计压缩至 30 分钟，AI 生成初稿 + 顾问审核微调 |
| 技术架构 | 前端 React + 后端 Claude API + Web Search |

## 2. 背景与问题

### 2.1 行业背景

国际高中留学咨询行业中，顾问在签约阶段需要为每位学生制定个性化的活动规划方案，包含课外活动、科研项目、实习、夏校等多个维度。这份方案是签约决策的关键交付物，直接影响客户转化率。

### 2.2 核心痛点

| 痛点 | 描述 | 影响 |
|------|------|------|
| 时间成本高 | 每份方案需 2-3 小时，涉及信息整理、活动设计、夏校匹配、时间线排布等多个环节 | 顾问产能受限，高峰期难以应对 |
| 质量不稳定 | 方案质量依赖顾问个人经验，新顾问难以达到资深顾问的水平 | 客户体验不一致，机构口碑波动 |
| 知识复用低 | 顾问的方法论存在于脑子里，难以规模化传承 | 新人培养周期长，团队扩张困难 |
| 信息检索低效 | 夏校、竞赛信息需要手动查找，且信息可能过时 | 方案中可能推荐已停办的夏校或过时信息 |

## 4. 方法论模型

以下是产品内置的核心方法论，源自资深留学顾问的实战经验，经过结构化整理后作为 AI 生成的基础逻辑。

### 4.1 整体流程（5 Steps）

| 步骤 | 名称 | 描述 | 执行方 |
|------|------|------|--------|
| Step 1 | 信息采集 | 收集学生背景信息：年级、学校、科目偏好、MBTI、兴趣爱好、好奇心、已有活动、父母职业等 | 顾问手动录入 |
| Step 2 | 专业定向 | 确定 2-3 个专业方向 | 顾问手动输入（V2 版本 AI 接管） |
| Step 3 | 提炼 PT & PG | 生成 Personal Tag（个人特色标签）和 Personal Goal（个人目标） | AI 生成 + 顾问审核 |
| Step 4 | 活动设计 | 围绕 PT 和 PG 设计 10 项活动 | AI 生成 + 顾问审核 |
| Step 5 | 时间规划 | 按月排布活动时间线，从当前年级到 12 年级 | AI 生成 + 顾问审核 |

### 4.2 Personal Tag（PT）

- **定义**：与专业无关的个人特色标签，作为活动主线和主文书素材来源
- **来源**：从学生的兴趣爱好、特长、个人经历中提取
- **示例**：
  - "冰球队长"（许業恒 - 建筑学）
  - "书法绘画 + 文创设计"（彭婉婷 - 化学工程）
  - "胶卷摄影 + 大提琴"（刘盈含 - 动物科学）
  - "嘴哈音乐"（虞哲 - 传媒学）
  - "旅行摄影 + 极光爱好"（陈锦坤 - 应用数学）
- **质量标准**：具体而生动，能一句话让人记住这个学生的特色

### 4.3 Personal Goal（PG）

- **定义**：与专业相关的个人目标，回答"为什么要学这个专业"
- **范围要求**：不能过于宽泛（如"改善世界"），也不能过于具体成为一个课题
- **示例**：
  - **彭婉婷**（化学工程/材料化学）：通过材料化学研究，探索可食用/可降解包装材料的开发，提升食品安全与环保可持续性
  - **许業恒**（建筑学）：研究历史建筑保护与城市更新的平衡，探索旧建筑改造与社区功能复兴的路径
  - **Kpop女生**（营养学/公共健康）：通过营养学/公共健康研究，了解青少年生长发育阶段的营养需求及极端饮食对健康的影响，探索青少年饮食失调的健康风险及其干预策略
- **生成要点**：PG 必须结合学生的好奇心和个人经历来推导，而非泛泛而谈

### 4.4 活动分类体系（Activity Taxonomy）

每份方案包含约 10 项活动，分布在以下六大类别中。各类别的数量可根据学生专业方向灵活调整。

| 类别 | 数量 | 设计原则 | 产出成果 |
|------|------|----------|----------|
| 科研项目 | 2 | ① 基础课题（围绕 PG）→ 论文；② 进阶课题（PG 延伸 / PG×PT 交叉 / 跨专业）→ 论文+项目成果 | 论文、项目、专利等 |
| 校内社团 | 2 | ① 专业相关社团（化学社、机器人社等）；② 兴趣类社团（音乐、艺术、体育、辩论等），争取 Leader / Core Member 角色 | Leader/核心成员身份 |
| 个人项目 | 1 | 尽量结合 PT × PG 交叉点，成果形式多样化 | App、网站、作品集、展览、绘本等 |
| 实习 | 1-2 | 利用假期进入专业相关公司/实验室，体验真实工作场景 | 实习经历、推荐信 |
| 实践活动 | 1-2 | ① 专业实践（强专业性）；② 志愿服务（专业特长×公益） | 项目成果、社会影响力 |
| 夏校 | 1 | 专业相关的高含金量夏校，能力突出者可考虑科研夏校（Pioneer/SSHI） | 学分、推荐信、研究经历 |

## 5. 功能需求

### 5.1 学生信息录入模块

| 字段名称 | 字段类型 | 是否必填 | 说明 |
|----------|----------|----------|------|
| 学生姓名（中/英） | 文本 | 必填 | 中文名 + 英文名 |
| 性别 | 单选 | 必填 | 男/女 |
| 当前年级 | 单选 | 必填 | 9/10/11/12 年级 |
| 学校名称 | 文本 | 必填 | 国际高中名称 |
| 课程体系 | 单选 | 必填 | IB / AP / A-Level / 其他 |
| 当前 GPA / 排名 | 文本 | 必填 | 成绩情况概览 |
| 标化考试目标 | 文本 | 选填 | 托福/雅思/SAT 目标分数 |
| MBTI | 单选 | 选填 | 16 种人格类型 |
| 喜欢的科目 | 多选 | 必填 | 包括擅长科目和感兴趣科目 |
| 兴趣爱好 | 多行文本 | 必填 | 详细描述兴趣爱好、特长 |
| 好奇心 / 关注领域 | 多行文本 | 必填 | 对什么问题感兴趣，想探索什么 |
| 已有活动 | 多行文本 | 选填 | 校内社团、实践经历等 |
| 父母职业背景 | 多行文本 | 选填 | 可能影响资源对接 |
| 专业方向（2-3个） | 文本 | 必填 | 顾问手动输入 |
| 申请国家/方向 | 多选 | 必填 | 美国 / 英国 / 加拿大 / 澳洲 / 其他 |

### 5.2 AI 生成模块

#### 5.2.1 PT & PG 生成

- **输入**：学生信息表全部字段 + 顾问输入的专业方向
- **输出**：1 个 PT（含简短描述）+ 每个专业方向各 1 个 PG
- **AI 策略**：提供 2-3 个备选方案供顾问选择，而非直接输出单一结果
- **质量控制**：PT 必须具体生动；PG 范围适中，不宽泛不狭窄

#### 5.2.2 活动方案生成

- **输入**：学生信息 + 确认后的 PT & PG
- **输出**：10 项活动，每项包含：编号、活动分类、活动名称、详细描述
- **生成规则**：
  - 科研课题①必须围绕 PG，课题②必须有延伸或交叉
  - 社团一个专业相关、一个兴趣相关
  - 个人项目必须体现 PT × PG 的交叉点
  - 实习必须对应真实可行的行业/公司场景
  - 实践活动区分专业实践和志愿服务两个方向
  - 夏校通过 Web Search 匹配当前有效的项目
- **调整规则**：实验类专业可增加 1-2 个实验室经历，相应减少实习或实践活动

#### 5.2.3 夏校推荐生成

- **实现方式**：AI 通过 Web Search 实时检索专业相关夏校
- **输出内容**：夏校名称、项目介绍、官网链接、申请截止日期、费用
- **顾问职责**：验证夏校是否仍在运行，确认信息准确性

#### 5.2.4 时间规划生成

- **输入**：确认后的 10 项活动 + 学生当前年级
- **输出格式**：
  - 横轴 6 列：学术成绩、校内活动、竞赛比赛、教授科研、校外实践、申请准备
  - 纵轴：按月排列，从当前年级到 12 年级
- **规划原则**：
  - 学期中优先校内活动和竞赛，假期集中安排实习/夏校/科研强度工作
  - 重要考试前后需留出复习/冲刺时间，其他活动适当让步
  - 科研项目按"选题→文献→研究→论文"节奏分阶段排布
  - 12 年级上学期集中处理申请材料，活动基本收尾

### 5.3 顾问审核模块

- **在线编辑**：顾问可直接修改 AI 生成的 PT、PG、活动描述和时间规划
- **重新生成**：对不满意的单项活动可触发重新生成，保留其他内容不变
- **备注功能**：顾问可为每项活动添加内部备注（不出现在导出文件中）

### 5.4 方案导出模块

- **导出格式**：Excel（.xlsx）+ Word（.docx）

**Excel 内容（4 个 Sheet）：**

| Sheet | 内容 |
|-------|------|
| Sheet 1: Activity List | 编号、活动分类、Activity、Detailed Description |
| Sheet 2: Honor | 竞赛规划和奖项目标 |
| Sheet 3: 可参加竞赛介绍 | 竞赛名称、简介、内容、资格、考试、时间、费用、奖项、官网 |
| Sheet 4: 夏校推荐 | 夏校名称、项目介绍、线上/线下、官网、时长、适合年级、截止日期、申请材料 |

**Word 内容：**
- 学生基础信息卡片（专业方向、能力模型、性格特点）
- 按月时间规划表（6 列 × N 月）

## 6. 信息架构与用户流程

### 6.1 页面结构

| 页面 | 功能描述 | 交互重点 |
|------|----------|----------|
| 登录页 | 顾问账号登录 | 简洁，支持邮箱登录 |
| 学生列表页 | 已创建的学生方案总览 | 卡片式布局，显示姓名、专业、状态 |
| 学生信息录入页 | 结构化表单填写学生背景信息 | 分步式表单，降低认知负荷 |
| PT & PG 生成页 | AI 生成备选方案，顾问选择/编辑 | 左右对比布局，支持快速切换 |
| 活动方案页 | 10 项活动的生成、展示和编辑 | 卡片式展示，单项可重新生成 |
| 时间规划页 | 按月时间线生成和编辑 | 表格式展示，单元格可编辑 |
| 方案导出页 | 预览和导出最终方案文件 | 一键导出 Excel + Word |

### 6.2 核心用户流程

```
顾问登录
  → 创建新学生
  → 填写学生信息表单
  → 输入 2-3 个专业方向
  → AI 生成 PT & PG（顾问选择/编辑）
  → AI 生成 10 项活动方案（顾问审核/微调）
  → AI 生成月度时间规划（顾问审核/微调）
  → 导出最终方案（Excel + Word）
```

### 6.3 状态流转

```
信息录入中 → 待生成 → PT/PG 待确认 → 活动待审核 → 时间规划待审核 → 已完成
```

> 顾问可以在任何阶段回退到上一步进行修改。

## 7. 技术架构

### 7.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React + TailwindCSS | 组件化 UI，响应式布局 |
| AI 引擎 | Claude API（Sonnet） | 方案生成的核心 LLM |
| 搜索能力 | Web Search API | 夏校实时检索与匹配 |
| 后端 | Node.js / Python | API 服务、数据处理 |
| 数据库 | PostgreSQL / Supabase | 学生信息和方案存储 |
| 文件生成 | docx-js + xlsx 库 | 导出 Excel 和 Word 文件 |

### 7.2 AI Prompt 架构

AI 生成采用 **System Prompt + Few-shot Examples + 结构化输出**的方式。

| 组件 | 内容 |
|------|------|
| System Prompt | 内置方法论规则、活动分类体系、PT/PG 定义和质量标准 |
| Few-shot Examples | 5 个真实案例（彭婉婷、许業恒、刘盈含、虞哲、陈锦坤） |
| 输出格式 | 结构化 JSON，便于前端渲染和文件导出 |
| 温度设置 | Temperature 0.7（平衡创造性和稳定性） |

## 9. 成功指标

### 9.1 核心指标

| 指标 | 定义 | MVP 目标 |
|------|------|----------|
| 方案生成时间 | 从输入学生信息到导出完成的端到端时间 | ≤ 30 分钟（从 2-3 小时降至） |
| AI 初稿采纳率 | AI 生成的活动中，顾问审核后保留不变的比例 | ≥ 60% |
| 顾问满意度 | 顾问对生成方案质量的主观评分（1-5） | ≥ 4.0 |
| 周方案产出 | 顾问每周完成的方案数量 | 提升 50%+ |

### 9.2 风险与应对

| 风险 | 等级 | 应对策略 |
|------|------|----------|
| AI 生成质量不稳定 | 高 | 多备选方案 + 顾问强制审核环节 + 持续优化 Prompt |
| 夏校信息过时 | 中 | Web Search 实时检索 + 顾问人工验证环节 |
| 方法论覆盖不全 | 中 | 用 5 个多样化案例训练，持续收集新案例迭代 |
| 顾问使用习惯迁移 | 低 | 提供引导式 Onboarding，降低学习成本 |
